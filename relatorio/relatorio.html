Relatório sobre Frameworks e Desenvolvimento Web

Projeto WSIP - What Should I Play?

Aluno: Bruno da Silva Martins, Felipe da Silva Paz

1. O que são Frameworks e para que são utilizados?

Frameworks são conjuntos de ferramentas, bibliotecas e estruturas pré-definidas que auxiliam no desenvolvimento de aplicações de software. Eles fornecem uma base organizada para que os desenvolvedores possam criar sistemas de forma mais rápida, segura e padronizada, evitando a necessidade de implementar funções comuns do zero.

No desenvolvimento web, os frameworks são amplamente utilizados para organizar o código, facilitar a comunicação com bancos de dados, gerenciar rotas, implementar mecanismos de segurança e agilizar a criação de interfaces e funcionalidades. Dessa forma, aumentam a produtividade e tornam a manutenção do sistema mais prática.

2. Frameworks para Programação Web (Back-end)

Linguagem Framework Propósito Diferencial
Python FastAPI Desenvolvimento de APIs modernas e de alta performance. Documentação automática e excelente desempenho.
Java Javalin Criação simplificada de aplicações web e APIs. Facilidade de aprendizado e configuração mínima.
PHP CodeIgniter Desenvolvimento rápido de aplicações web. Leveza e baixo consumo de recursos.
JavaScript Fastify Criação de APIs e aplicações backend em Node.js. Alto desempenho em requisições simultâneas.

3. Framework Utilizado no Projeto

Para o desenvolvimento da aplicação foi utilizado o framework Flask, escrito em Python.

O Flask é um microframework voltado para aplicações web. Sua principal característica é a simplicidade, permitindo que o desenvolvedor tenha maior controle sobre a estrutura do sistema.

A escolha do Flask ocorreu devido à sua facilidade de uso, boa documentação e integração simples com banco de dados SQLite e APIs externas e principalmente pela recomendação do professor de programação Henrique Sant'Anna.

4. Descrição do Projeto Desenvolvido

Nome do Projeto

WSIP - What Should I Play?

Objetivo

O objetivo do projeto é auxiliar jogadores que possuem muitos jogos em suas bibliotecas e encontram dificuldade para decidir qual deles jogar.

O sistema permite cadastrar jogos, armazenar informações automaticamente, acompanhar o progresso de cada título e realizar sorteios aleatórios para recomendar uma próxima experiência ao usuário.

5. Funcionalidades Implementadas

  • Cadastro de jogos.
  • Exibição da biblioteca de jogos.
  • Atualização de status dos jogos.
  • Exclusão de registros.
  • Consulta automática de informações através de API.
  • Exibição automática de capas dos jogos.
  • Sorteio aleatório de um jogo da biblioteca.
  • Persistência dos dados utilizando banco SQLite.
  • Interface web desenvolvida em HTML e CSS.

6. Tecnologias Utilizadas

Tecnologia Finalidade
Python Linguagem principal do sistema.
Flask Framework backend.
SQLite Persistência de dados.
HTML Estrutura das páginas.
CSS Estilização da interface.
RAWG API Obtenção automática de informações dos jogos.

7. Requisitos da Atividade Atendidos

  • Utilização de Framework Web (Flask).
  • CRUD completo (Create, Read, Update e Delete).
  • Consumo de API realizado no backend.
  • Processamento de dados através do sorteio de jogos.
  • Persistência utilizando banco de dados SQLite.
  • Renderização de páginas HTML por meio de templates.
  • Versionamento utilizando Git e GitHub.

9. Conclusão

O desenvolvimento deste trabalho permitiu aplicar conceitos importantes relacionados ao uso de frameworks, desenvolvimento web, integração com APIs externas e persistência de dados.
